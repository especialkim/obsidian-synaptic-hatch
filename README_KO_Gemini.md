# Synaptic Hatch for Obsidian

Obsidian을 다른 모든 애플리케이션 위에 항상 고정하여, 글을 쓰거나 다른 작업을 하면서도 노트를 빠르게 참조하거나 메모할 수 있는 작업 공간을 만들어주는 플러그인입니다.

## 주요 기능

- **항상 위 고정 (Always-on-Top Pinning)**: Obsidian의 메인 창이나 팝아웃 창을 화면의 다른 모든 창들 위에 항상 떠 있도록 고정할 수 있습니다.
- **고정 상태 표시기 (Pin Indicator)**: 창이 고정되었는지 한눈에 알 수 있도록 핀 모양의 아이콘을 표시합니다. 이 표시기는 메인 창과 팝아웃 창에 대해 각각 다른 위치와 크기로 설정할 수 있습니다.
- **사용자 정의 팝아웃 명령어 (Custom Popout Commands)**: 자주 사용하는 노트나 작업 흐름에 맞춰 '항상 위 고정' 팝아웃 창을 여는 자신만의 명령어를 만들 수 있습니다. 예를 들어, 특정 파일을 열거나, 오늘 날짜의 저널 노트를 바로 여는 등의 명령어를 설정할 수 있습니다.

## 사용 방법

### 명령어 팔레트 (Command Palette)

`Cmd/Ctrl + P`를 눌러 명령어 팔레트를 열고 다음 명령어들을 사용할 수 있습니다.

- **`Toggle Window Pin for Always on Top`**: 현재 활성화된 Obsidian 창(메인 또는 팝아웃)의 '항상 위 고정' 상태를 켜거나 끕니다.
- **`Open Always-On-Top Popout`**: 현재 보고 있는 노트를 '항상 위 고정'이 적용된 새로운 팝아웃 창으로 엽니다.
- **`Custom Popout > ...`**: 사용자가 직접 설정한 '사용자 정의 팝아웃 명령어'가 이 이름으로 표시됩니다.

### 핀 표시기 (Pin Indicator)

창의 가장자리에 나타나는 핀 모양 아이콘을 클릭하여 '항상 위 고정' 상태를 간편하게 켜고 끌 수 있습니다.

## 설정

플러그인 설정 탭에서 'Synaptic Hatch'를 찾아 다양한 옵션을 설정할 수 있습니다.

### 1. Main Window Indicator (메인 창 표시기)

메인 Obsidian 창에 표시되는 핀 아이콘의 스타일과 동작을 설정합니다. (기본적으로는 비활성화되어 있으며, 창이 고정되었을 때만 나타납니다.)

- **`Show indicator in main window`**: 메인 창에 핀 표시기를 항상 표시할지 여부를 결정합니다.
- **`Top offset (px)`**: 창 상단에서의 거리
- **`Right offset (px)`**: 창 우측에서의 거리
- **`Indicator size (px)`**: 표시기 전체 크기
- **`Icon size (px)`**: 표시기 내부 아이콘 크기

### 2. Pop-out Window Indicator (팝아웃 창 표시기)

팝아웃 창에 표시되는 핀 아이콘의 스타일을 설정합니다. (기본적으로 활성화되어 있습니다.)

- **`Show indicator in popout windows`**: 팝아웃 창에 핀 표시기를 표시할지 여부를 결정합니다.
- 나머지 설정은 메인 창 표시기와 동일합니다.

### 3. Custom Popout Commands (사용자 정의 팝아웃 명령어)

이 플러그인의 가장 강력한 기능 중 하나입니다. 특정 조건으로 '항상 위 고정' 팝아웃 창을 여는 자신만의 명령어를 만들 수 있습니다.

- **`Use Custom Popout Commands`**: 이 기능을 활성화해야 아래의 명령어들을 생성하고 사용할 수 있습니다.
- **`+ Add Command`**: 새로운 사용자 정의 명령어를 추가합니다.

#### 명령어 유형 (Command Type)

각 명령어는 다음 네 가지 유형 중 하나를 선택할 수 있습니다.

- **`Blank`**: 비어있는 새 노트를 팝아웃 창으로 엽니다.
- **`File`**: 지정한 특정 파일을 팝아웃 창으로 엽니다.
  - `File path`: 팝아웃으로 열고 싶은 파일의 경로를 입력합니다.
- **`Folder`**: 지정된 폴더 안에 새로운 노트를 생성하고 그 노트를 팝아웃 창으로 엽니다.
  - `Folder path`: 새 노트를 생성할 폴더의 경로를 입력합니다. (비워두면 Vault 최상위)
  - `Filename rule`: 생성될 파일의 이름을 지정합니다. `{{date}}`를 사용하여 오늘 날짜를 파일명에 포함시킬 수 있습니다. (예: `회의록-{{date}}`)
  - `Template path`: 새 노트 생성 시 사용할 템플릿 파일의 경로를 지정할 수 있습니다.
- **`Journal`**: 'Periodic Notes' 플러그인과 연동하여 일간/주간/월간 노트를 팝아웃 창으로 엽니다.
  - `Journal Period`: 열고 싶은 저널의 주기(daily, weekly 등)를 선택합니다.

#### 명령어 활성화 및 사용

1.  원하는 유형과 세부 설정을 마친 후, 명령어 오른쪽에 있는 **토글 스위치**를 켜서 활성화해야 합니다.
2.  설정이 유효하지 않은 경우(예: 파일 경로가 잘못된 경우) 토글이 켜지지 않고 오류 알림이 표시됩니다.
3.  활성화된 명령어는 명령어 팔레트(`Cmd/Ctrl+P`)에 `Custom Popout > ...` 형태로 나타납니다.
4.  **URI 복사 버튼**을 클릭하면 해당 명령어를 실행할 수 있는 `obsidian://` 형태의 URI가 복사됩니다. 이 URI를 다른 앱이나 스크립트에서 사용하여 해당 팝아웃을 바로 열 수 있습니다.

### 4. Miscellaneous (기타 설정)

- **`Date format`**: 'Folder' 타입의 명령어에서 `{{date}}` 변수를 사용할 때 적용될 날짜 형식을 지정합니다. (예: `YYYY-MM-DD`)

## 설치 방법

1.  Obsidian의 '설정' > '커뮤니티 플러그인'으로 이동합니다.
2.  '안전 모드'가 꺼져 있는지 확인합니다.
3.  '커뮤니티 플러그인 찾아보기'를 클릭합니다.
4.  `Synaptic Hatch`를 검색하여 설치합니다.
5.  설치 후 플러그인을 활성화합니다.
